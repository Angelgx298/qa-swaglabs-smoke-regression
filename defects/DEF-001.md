**Severity**: Low  
**Priority**: P3  
**Module**: Inventory  
**Status**: Open  
**Found**: Nov 2025 (Manual exploratory testing)

## Description

The product sort dropdown on inventory page does not have an explicit `<label>` element or `aria-label` attribute. It relies only on CSS class naming, which may create issues for screen reader users.

## Steps to Reproduce

1. Navigate to https://www.saucedemo.com/
2. Login with standard_user
3. Inspect the sort dropdown with browser DevTools
4. Check for associated label element or aria-label

## Actual Result

- No `<label for="...">` element present
- No `aria-label` attribute on select element
- Element identified only by class `.product_sort_container`

## Expected Result

Should have explicit label like:

- `<label for="product_sort_container">Sort products by</label>`, OR
- `<select aria-label="Sort products" ...>`

## Impact

- **Accessibility**: Violates WCAG 2.1 Level A (4.1.2 Name, Role, Value)
- **User Impact**: Screen reader users may have difficulty identifying the control
- **Workaround**: Visual users not affected

## Evidence

Browser: Chrome 131, Firefox 133  
URL: https://www.saucedemo.com/inventory.html

## Suggested Fix

Add `aria-label="Sort products"` to the select element or wrap with proper label element.
