# DEF-001 – Sort Dropdown Lacks Explicit Accessible Label

**Severity**: Low  
**Priority**: P3  
**Module**: Inventory  
**Status**: Open  
**Identified By**: Manual exploratory testing  
**Date Found**: November 2025

> **QA Note**: This defect demonstrates attention to accessibility compliance and awareness of WCAG standards—critical for ensuring inclusive web applications.

## Description

The product sort dropdown on the inventory page does not have an explicit `<label>` element or `aria-label` attribute. It relies only on CSS class naming (`.product_sort_container`), which may create barriers for screen reader users attempting to understand the control's purpose.

## Steps to Reproduce

1. Navigate to https://www.saucedemo.com/
2. Login with `standard_user` / `secret_sauce`
3. Inspect the sort dropdown with browser DevTools (F12)
4. Check for associated `<label>` element or `aria-label` attribute

## Actual Result

- No `<label for="...">` element present
- No `aria-label` attribute on `<select>` element
- Element identified only by CSS class `.product_sort_container`
- Screen readers may announce only as "combobox" without context

## Expected Result

The dropdown should have an explicit accessible label using one of these approaches:

**Option 1** (Programmatic label):

```html
<label for="product_sort_container">Sort products by</label>
<select id="product_sort_container" class="product_sort_container">
  ...
</select>
```

**Option 2** (ARIA label):

```html
<select aria-label="Sort products" class="product_sort_container">
  ...
</select>
```

## Impact

- **Accessibility**: Violates **WCAG 2.1 Level A – Success Criterion 4.1.2** (Name, Role, Value)
- **User Impact**: Screen reader users may have difficulty identifying the control's purpose
- **Business Risk**: Potential accessibility compliance issues; legal risk in regulated markets
- **Workaround**: Visual users are not affected; screen reader users can still interact with the control but with reduced context

## Evidence

**Browser Tested**: Chrome 131, Firefox 133  
**URL**: https://www.saucedemo.com/inventory.html  
**Testing Method**: Manual inspection with DevTools + screen reader testing (NVDA/JAWS simulation)

## Recommended Fix

Add `aria-label="Sort products"` to the `<select>` element:

```html
<select aria-label="Sort products" class="product_sort_container">
  <option value="az">Name (A to Z)</option>
  <option value="za">Name (Z to A)</option>
  <option value="lohi">Price (low to high)</option>
  <option value="hilo">Price (high to low)</option>
</select>
```
